隐私政策

生效日期：2026年6月16日

版本：1.0

1) 控制者

个人数据的控制者为 Canario-Lab / ScreenClips 应用。

ScreenClips 是一款在本地运行的 iOS 应用，用于把截图整理成 Clips 和资料库。使用本应用无需用户账号或注册。

2) 本地存储的数据

在应用中创建的数据，包括 Clips、资料库、备注、图标、截图、语言设置、Face ID 设置以及备份相关信息，会存储在用户设备本地，或存储在主应用与分享扩展共同使用的本地 App Group 容器中。

这些数据不会自动发送给 Canario-Lab 或 Canario-Lab 的服务器。

3) 照片、截图和分享扩展

应用仅在将用户选定的图片添加到 Clip 所需范围内访问“照片”。ScreenClips 会保存一份针对应用内使用优化过的截图副本。

从 Clip 中移除截图时，只会从 ScreenClips 的数据中移除。用户“照片”图库中的原始图片不会被删除。

分享扩展允许用户从系统分享面板把选定图片发送到 ScreenClips。该扩展通过 App Group group.com.canario-lab.screenclip.shared 将数据写入同一本地存储。

4) 备份、导出和分享

备份会创建一个本地 .screenclipbackup 文件，其中包含 Clips、资料库、截图和应用设置。该文件会通过用户选择的系统功能保存或分享。

从备份恢复会用所选文件中的数据替换应用当前的本地状态。用户负责妥善保存和保护导出的备份、PDF 以及从应用外分享出去的其他材料。

5) Face ID 和设备安全

如果用户启用 Face ID 应用锁，身份验证由 iOS 通过 LocalAuthentication 处理。ScreenClips 不接收、不存储生物识别数据。

由于应用数据保存在本地，使用设备密码、Face ID 或其他系统安全机制保护设备本身仍然很重要。

6) Premium 购买、Apple 和 RevenueCat

ScreenClips 可能在 App Store 提供 Premium，形式为一次性 Lifetime 购买。应用不提供月度或年度订阅，也不提供试用期。

付款由 Apple 通过 App Store 和 StoreKit 处理。ScreenClips 使用 RevenueCat 展示购买选项、验证 ScreenClips Pro 权益、恢复购买，并在应用中显示技术性的 RevenueCat ID。

在此范围内，Apple 和 RevenueCat 可能处理技术性购买数据，例如交易标识符、产品标识符、Premium 访问状态，以及用于确认访问权限的匿名应用用户标识符。

Canario-Lab 不处理用户的支付卡信息。

7) 数据接收方

只有在用户选择相关功能或处理购买所必需的范围内，数据才可能由外部服务处理：

  • Apple，用于 App Store、StoreKit、付款、系统文件选择器、照片、Face ID 和分享。
  • RevenueCat，用于展示购买选项、验证 Premium 和恢复 Lifetime 购买。
  • 用户自行选择的系统服务，当用户将 PDF、备份或其他文件导出或分享到 ScreenClips 之外时。
  • 公共机关，在适用法律要求时。

8) 保留期限

保存在 ScreenClips 中的数据会留在本地，直到用户删除数据、恢复其他备份、移除应用，或通过 iOS 系统机制移除数据。

导出的备份、PDF 和分享到应用外的文件，由用户以及接收这些文件的服务控制。

由 Apple 和 RevenueCat 处理的购买数据，按照这些服务各自的政策保留。

9) 用户权利

如果 Canario-Lab 在具体场景中处理用户个人数据，例如支持联系，用户可以请求访问、更正、删除、限制处理、反对处理，并向有管辖权的监管机构提出投诉。

如有隐私相关问题，请使用 App Store 或应用中列出的联系方式。

10) 本政策的变更

当 ScreenClips 的功能、付款模式或法律要求发生变化时，本隐私政策可能会更新。当前版本将在应用中提供。
