Privacy Policy

Effective date: June 16, 2026

Version: 1.0

1) Data controller

The controller of personal data is Canario-Lab / ScreenClips app.

ScreenClips is a local iOS app for organizing screenshots into Clips and libraries. The app does not require a user account or registration.

2) Data stored locally

Data created in the app, including Clips, libraries, notes, icons, screens, language settings, Face ID settings, and backup information, is stored locally on the user's device or in the local App Group container shared by the main app and the share extension.

This data is not automatically sent to Canario-Lab or to Canario-Lab servers.

3) Photos, screens, and Share Extension

The app uses Photos access only to add images selected by the user to a Clip. ScreenClips stores its own optimized copy of the screen for use inside the app.

Removing a screen from a Clip removes it only from ScreenClips data. It does not delete the original photo from the user's Photos library.

Share Extension lets the user send selected images from the system share sheet to ScreenClips. The extension writes data to the same local store through the App Group group.com.canario-lab.screenclip.shared.

4) Backups, export, and sharing

A backup creates a local .screenclipbackup file containing Clips, libraries, screens, and app settings. The file is saved or shared through system mechanisms selected by the user.

Restoring a backup replaces the current local app state with data from the selected file. The user is responsible for storing and protecting exported backup files, PDFs, and any other materials shared outside the app.

5) Face ID and device security

If the user enables Face ID app lock, authentication is handled by iOS through LocalAuthentication. ScreenClips does not receive or store biometric data.

Because app data is stored locally, protecting the device itself with a passcode, Face ID, or other system security mechanisms remains an important part of security.

6) Premium purchases, Apple, and RevenueCat

ScreenClips may offer paid Premium access as a one-time Lifetime purchase in the App Store. The app does not offer a monthly or yearly subscription and does not offer a trial.

Payments are handled by Apple through the App Store and StoreKit. ScreenClips uses RevenueCat to present the offer, verify the ScreenClips Pro entitlement, restore the purchase, and show the technical RevenueCat ID in the app.

In this scope, Apple and RevenueCat may process technical purchase data such as transaction identifiers, product identifiers, Premium access status, and an anonymous app user identifier used to confirm access.

Canario-Lab does not process the user's payment card details.

7) Data recipients

Data may be processed by external services only to the extent needed for features selected by the user or purchase handling:

  • Apple, for App Store, StoreKit, payments, system file pickers, Photos, Face ID, and sharing.
  • RevenueCat, for offer handling, Premium verification, and restoring the Lifetime purchase.
  • System providers selected by the user, if the user exports a PDF, backup, or shares a file outside ScreenClips.
  • Public authorities, if required by applicable law.

8) Retention period

Data saved in ScreenClips remains local until the user deletes it, restores another backup, removes the app, or removes data through iOS system mechanisms.

Exported backups, PDFs, and files shared outside the app remain under the control of the user and the services where they were sent.

Purchase data handled by Apple and RevenueCat is retained according to the policies of those services.

9) User rights

If Canario-Lab processes the user's personal data in a specific case, for example through support contact, the user may request access, rectification, deletion, restriction of processing, object to processing, and lodge a complaint with the competent supervisory authority.

For privacy matters, use the contact channel listed in the App Store or in the app.

10) Changes to this policy

This Privacy Policy may be updated along with changes to ScreenClips features, the payment model, or legal requirements. The current version will be available in the app.
